# Project Editor Agent

This folder owns specialized guidance for agents drafting, reviewing, or revising portfolio project content.

Use these docs with [Agent Workflow Playbooks](../agent-workflow-playbooks.md). The playbook remains the router for repo boundaries, files to inspect, admin import shape, verification, and stop conditions. This folder owns the editorial decision model for case-study content.

## Current Docs

- [Case Study Draft Guidelines](./case-study-draft-guidelines.md): audience model, field limits, anti-bloat rules, outlier criteria, and review checklist for project card and modal copy.

## Maintenance Rules

- Keep this folder focused on project editing guidance.
- Prefer one durable rule over repeated examples of the same rule.
- Update these docs when modal rendering, audience strategy, supported project content fields, or case-study quality standards change.
- Keep project-specific audits and one-time review notes in planning folders unless they become durable rules.
